IMPLEMENT THESE 4 CORE PLANNER RULES
==================================================

RULE 1 — Discovery rule
If the mission has an entity + location, always begin with candidate discovery.

Default discovery step:
SEARCH_PLACES

Purpose:
Get candidate entities first before deeper verification.

Examples:
- pubs in Arundel
- dentists in Brighton
- organisations in Blackpool

==================================================

RULE 2 — Direct field check rule
If a constraint can be checked directly against known structured fields, do not send the run into generic website/evidence logic first.

Examples:
- text_compare on name
- simple direct attribute/category/class checks that can be checked from search/discovery results

Plan shape:
SEARCH_PLACES
→ FILTER_FIELDS / MATCH_DIRECT_FIELDS

Examples:
- “pubs in Arundel with Swan in the name”
- “dentists in Brighton with clinic in the name”

Important:
This rule should avoid unnecessary web visits.

==================================================

RULE 3 — Website evidence rule
If the constraint requires evidence from the organisation’s own site, use website verification after discovery.

Examples:
- mentions live music on website
- says vegan on site
- mentions private dentistry on its website

Plan shape:
SEARCH_PLACES
→ WEB_VISIT
→ EVIDENCE_EXTRACT
→ TOWER_JUDGE

Applies mainly to:
- website_evidence
- some status_check
- some attribute_check

==================================================

RULE 4 — Relationship / external evidence rule
If the constraint is about a relationship that is not directly present in Places-style discovery data, it must never compile to SEARCH_PLACES-only.

Examples:
- works with local authority
- supplies pubs
- partners with universities
- funded by council

Plan shape:
SEARCH_PLACES
→ WEB_SEARCH
→ WEB_VISIT
→ EVIDENCE_EXTRACT
→ TOWER_JUDGE

Critical requirement:
relationship_check must never silently degrade into plain discovery-only planning.

This is the key Blackpool failure we are fixing.
