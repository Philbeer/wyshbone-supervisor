PART 3 — FAILURE CLASSIFICATION LOGIC

Create a classifier utility:

/src/evaluator/classifyRunFailure.ts

Function:

classifyRunFailure(runContext, towerVerdict, planHistory)

Goal:

Return the most likely failure bucket.

Rules (simple heuristics):

1. If mission parsing failed or constraints invalid
→ INTERPRETATION_FAILURE

2. If plan generated but obviously empty or invalid
→ PLANNER_FAILURE

3. If candidate pool < requested results / 2
→ DISCOVERY_FAILURE

4. If crawler failed or returned empty pages
→ CRAWL_FAILURE

5. If pages fetched but evidence items are weak or missing quotes
→ EVIDENCE_EXTRACTION_FAILURE

6. If strong evidence exists but Tower rejected it
→ TOWER_JUDGEMENT_FAILURE

7. If Tower returned CHANGE_PLAN but plan v2 is nearly identical
→ REPLAN_FAILURE

8. If Tower verdict differs from what UI presents
→ UI_TRUTH_FAILURE

Otherwise:

→ NONE

Keep the classifier simple and deterministic.
