ask: Produce a diagnostic report covering the last 14 completed runs in Wyshbone.

Purpose: Identify what actually happened in each run, where decisions were made, and where behaviour diverged from expectations. This is for internal truth-finding, not user-facing output.

For EACH run, report the following fields clearly and consistently:

Run identifiers

run_id

client_request_id (CRID)

timestamp

User input

Raw user prompt exactly as received

Parsed intent (what the system believed the user wanted)

Constraints

List all constraints extracted

Classify each as: HARD / SOFT

Note any constraints marked unverifiable or ambiguous

Plan evolution

Plan versions executed (v1, v2, v3…)

For each plan:

What changed vs previous plan

Why it changed (Tower judgement, CVL, lack of results, etc.)

Search & execution behaviour

Search strategy used (location, radius, query terms, filters)

Any narrowing or broadening decisions made autonomously

Any unexpected or non-default strategy choices

Results

Requested count vs delivered count

Leads_list length

CVL verified_exact count

CVL hard_unknown count

Final outcome

Final verdict: PASS / FAIL / STOP

Stop reason if applicable (explicit, not implied)

Whether the delivery summary shown to the user matched the true outcome

Behavioural flags

Any of the following if they occurred:

Under-delivery without STOP

Over-claiming in delivery summary

Premature completion

Incorrect constraint relaxation

Search strategy mismatch (e.g. single-venue result in dense area)

Supervisor assessment

One-line judgement per run:

“Correct behaviour”

or “Technically correct but misleading”

or “Incorrect behaviour”

Cross-run patterns

Recurring failure modes across the 14 runs

Any systematic bias or repeated decision error

Rules

Do not optimise, justify, or soften outcomes

If something is unknown, state “unknown”

Prefer explicit counts and facts over narrative

This report is for fixing the system, not defending it

Output format

One section per run

Followed by a final “Systemic Findings” summary
