Implement Wyshbone Agent Loop v1 Phase 1 (minimal Tower-governed replan loop) in this Supervisor repo.

Goal: After a tool run produces an artefact and Tower returns a verdict, Supervisor must automatically react:
- ACCEPT -> emit afr event tower_judgement_received + run_completed (if not already), and post a plain "delivered vs requested / gaps / confidence" summary artefact (type=run_summary).
- RETRY -> retry the same tool once (maxRetry=1). If retry already used, STOP.
- CHANGE_PLAN -> create and emit a plan_update artefact (plan_version=2) and rerun with adjusted parameters (simple adjustment rules below).
- STOP -> emit run_stopped with reasons and post run_summary artefact.

Scope: do this only for SEARCH_PLACES first (one tool only). Do not touch CRM, email, or multi-run learning.

Implementation constraints:
- Keep it minimal, production safe.
- Add RunState per runId in memory (ok for now).
- Add retry limit: max 1 retry total.
- Add plan version limit: max plan_version=2.
- Tower verdict schema (v1):
  verdict: ACCEPT|RETRY|CHANGE_PLAN|STOP
  delivered: number or short summary
  requested: number or short summary
  gaps: string[]
  confidence: 0-100
  rationale: string
- After any verdict, always emit afr event: tower_judgement_received with verdict + confidence + gapsCount.
- Always post a run_summary artefact so UI can show Delivered vs Requested, gaps, confidence.

Adjustment rules for CHANGE_PLAN (simple):
- If delivered < requested: increase radius or broaden query terms by 1 step and rerun.
- If duplicate_rate too high: tighten filters and rerun.
If unsure, just broaden radius.
