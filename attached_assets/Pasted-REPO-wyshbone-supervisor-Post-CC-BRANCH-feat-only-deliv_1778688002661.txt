CONTEXT
After per-lead Tower semantic verification, each lead in mergedExact has a
match_valid flag indicating whether evidence actually supports the hard
constraints. Currently the final user-facing delivery uses all of mergedExact
regardless of match_valid, which means leads that Tower flagged as
"no_evidence" or "insufficient_evidence" still appear in the UI as
"verified results delivered" — misleading the user and making the system
look broken when those leads clearly don't match the constraint.

For example: "find micro pubs that opened in the last 6 months in sussex"
returns 2 candidates from GPT-4o web search, but per-lead Tower
verification finds no evidence either of them opened recently. Both have
match_valid=false. Yet the UI says "2 verified results delivered" and
shows them as results.

GOAL
Only deliver leads where per-lead match_valid is true. If 0 verified,
deliver 0 leads with an honest "no verified matches" message instead of
shipping unverified candidates.
