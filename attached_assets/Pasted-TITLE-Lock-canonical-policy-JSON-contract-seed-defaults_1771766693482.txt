TITLE: Lock canonical policy JSON contract + seed defaults (Learning Layer v1)

We are doing this properly. No shortcuts. You must standardise the policy JSON schema across the whole system so Supervisor, UI, and Tower cannot drift.

Requirements

Canonical schema
Make policy_versions.policy_json use this exact structure (no extra keys, no alternative names):

{
  "policy_bundle_version": 1,
  "policies": {
    "radius_policy_v1": {
      "enabled": true,
      "default_steps_km": [2, 5, 10],
      "max_cap_km": 10,
      "high_value_extra_step_km": 20,
      "min_new_eligible_to_continue": 2,
      "stop_if_last_n_runs_beyond_cap_added_less_than": 2,
      "last_n_runs_window": 5
    },
    "enrichment_policy_v1": {
      "enabled": true,
      "mode": "places_first_then_web",
      "max_enrich_calls_per_lead": 1,
      "max_total_enrich_calls": 25,
      "enrichment_batch_size": 10,
      "prefer_domains": [],
      "avoid_domains": [],
      "domain_reliability_seed": {}
    },
    "stop_policy_v1": {
      "enabled": true,
      "max_replans": 2,
      "search_budget_count": 1,
      "search_count": 1,
      "stop_when_verified_exact_is_zero_after_enrichment": true,
      "stop_when_cost_exceeds_budget": true,
      "known_unverifiable_hard_constraints": [
        "c_attr_live_music",
        "c_attr_opened_recently"
      ],
      "require_user_override_to_attempt_unverifiable_hard": true
    }
  }
}

Canonical policy application snapshot
Every run must write policy_applications.policies_applied_json in this exact structure:

{
  "scope_key": "vertical|geo_bucket|constraint_bucket",
  "applied_at": "ISO_TIMESTAMP",
  "applied_versions": {
    "radius_policy_v1": 1,
    "enrichment_policy_v1": 1,
    "stop_policy_v1": 1
  },
  "applied_policies": {
    "radius_policy_v1": {
      "default_steps_km": [2, 5, 10],
      "max_cap_km": 10
    },
    "enrichment_policy_v1": {
      "mode": "places_first_then_web",
      "max_total_enrich_calls": 25,
      "enrichment_batch_size": 10
    },
    "stop_policy_v1": {
      "max_replans": 2,
      "search_budget_count": 1,
      "search_count": 1,
      "stop_when_verified_exact_is_zero_after_enrichment": true,
      "require_user_override_to_attempt_unverifiable_hard": true
    }
  },
  "why_short": [
    "Default learning bundle applied (no learned updates yet)."
  ]
}

Default seeding (mandatory)
If there is no learned policy for a scope_key, Supervisor must deterministically fall back to a stored default bundle (not hard-coded only in memory). Do one of:

Store a reserved scope_key like GLOBAL_DEFAULT and read from it

Or auto-insert the default policy_versions rows on first use

Policy application must be real
Ensure the fields Supervisor already applies (searchBudgetCount, searchCount, MAX_REPLANS, enrichmentBatchSize, radius caps/steps) are pulled from this canonical JSON and actually affect execution.

No schema drift
If any existing stored policies use a different shape, write a small migration/compat layer to convert them into this canonical structure.

Definition of done

Every run produces a policy_applications row with the canonical snapshot JSON

Supervisor reads the canonical policy bundle and execution parameters change accordingly

Default bundle is stored and applied deterministically when no learned policy exists
