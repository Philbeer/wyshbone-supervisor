GOAL
Remove the hardcoded Anthropic Sonnet override in the chat handler so chat queries use the default LLM failover chain (Groq → Anthropic → OpenAI) defined in llm-failover.ts. With GROQ_API_KEY set in Secrets, this means chat will run on Llama 3.3 70B via Groq — substantially faster than Sonnet for chat-style queries.

WHY THIS IS SAFE
- The failover chain in llm-failover.ts will pick Groq first (since GROQ_API_KEY is set), then Anthropic, then OpenAI as fallback
- If Groq fails for any reason, it automatically falls back to Anthropic with the default Anthropic model (claude-3-5-haiku-20241022) — still fast, still works
- All other call sites that pass anthropicModel explicitly are unaffected — only the chat handler is changed
