What we're doing: The mission planner currently hardcodes tool sequences based on constraint types (e.g. status_check always triggers SEARCH_PLACES → WEB_VISIT → EVIDENCE_EXTRACT → TOWER_JUDGE). We're adding an LLM review step that can override this. The mission planner still classifies constraints (useful work), but GPT-4o-mini makes the final strategy/tool_sequence decision based on the full picture.
Step 1 — Create a new file: server/supervisor/reloop/strategy-reviewer.ts
typescript/**
 * Strategy Reviewer — LLM reviews the mission planner's strategy and can override it.
 * 
 * The mission planner classifies constraints and picks a default strategy using
 * hardcoded rules. This reviewer passes that classification + the intent narrative
 * to GPT-4o-mini and lets it confirm or override the strategy and tool sequence.
 * 
 * If the LLM call fails, the original plan is returned unchanged (safe fallback).
 */

import type { MissionPlan, PlanStrategyId, MissionToolStep } from '../mission-planner';
import type { IntentNarrative } from '../mission-schema';
import { createArtefact } from '../artefacts';

const VALID_STRATEGIES: PlanStrategyId[] = [
  'discovery_only',
  'discovery_then_direct_filter',
  'discovery_then_website_evidence',
  'discovery_then_rank',
];

const STRATEGY_TOOL_MAP: Record<string, MissionToolStep[]> = {
  discovery_only: ['SEARCH_PLACES'],
  discovery_then_direct_filter: ['SEARCH_PLACES', 'FILTER_FIELDS'],
  discovery_then_website_evidence: ['SEARCH_PLACES', 'WEB_VISIT', 'EVIDENCE_EXTRACT', 'TOWER_JUDGE'],
  discovery_then_rank: ['SEARCH_PLACES', 'RANK_SCORE'],
};

interface ReviewResult {
  reviewed: boolean;
  overridden: boolean;
  plan: MissionPlan;
  reasoning: string;
  latencyMs: number;
}

export async function reviewStrategy(params: {
  plan: MissionPlan;
  rawUserInput: string;
  intentNarrative: IntentNarrative | null;
  runId: string;
  userId: string;
  conversationId?: string;
}): Promise<ReviewResult> {
  const { plan, rawUserInput, intentNarrative, runId, userId, conversationId } = params;
  const startMs = Date.now();

  const openaiKey = process.env.OPENAI_API_KEY;
  if (!openaiKey) {
    console.warn('[STRATEGY_REVIEWER] No OPENAI_API_KEY — skipping review, using rules-based plan');
    return { reviewed: false, overridden: false, plan, reasoning: 'No API key', latencyMs: 0 };
  }

  try {
    const systemPrompt = buildPrompt(plan, rawUserInput, intentNarrative);

    const { default: OpenAI } = await import('openai');
    const client = new OpenAI({ apiKey: openaiKey });

    const timeoutMs = 10_000;
    const timeoutPromise = new Promise<never>((_, reject) =>
      setTimeout(() => reject(new Error('Strategy review timed out after 10s')), timeoutMs)
    );

    const completionPromise = client.chat.completions.create({
      model: 'gpt-4o-mini',
      messages: [
        { role: 'system', content: systemPrompt },
        { role: 'user', content: 'Review the strategy and respond with JSON only.' },
      ],
      temperature: 0.2,
      max_tokens: 256,
      response_format: { type: 'json_object' },
    });

    const response = await Promise.race([completionPromise, timeoutPromise]);
    const latencyMs = Date.now() - startMs;
    const rawContent = response.choices[0]?.message?.content ?? '';
    const usage = response.usage;

    console.log(`[STRATEGY_REVIEWER] Response received (latency=${latencyMs}ms, tokens=${usage?.prompt_tokens ?? '?'}/${usage?.completion_tokens ?? '?'})`);

    let parsed: { strategy: string; reasoning: string };
    try {
      parsed = JSON.parse(rawContent);
    } catch {
      console.warn(`[STRATEGY_REVIEWER] Failed to parse JSON: ${rawContent.substring(0, 200)}`);
      return { reviewed: true, overridden: false, plan, reasoning: 'JSON parse failed', latencyMs };
    }

    const chosenStrategy = parsed.strategy as PlanStrategyId;
    const reasoning = parsed.reasoning || 'No reasoning provided';

    if (!VALID_STRATEGIES.includes(chosenStrategy)) {
      console.warn(`[STRATEGY_REVIEWER] Invalid strategy "${chosenStrategy}" — keeping original`);
      return { reviewed: true, overridden: false, plan, reasoning: `Invalid strategy: ${chosenStrategy}`, latencyMs };
    }

    const overridden = chosenStrategy !== plan.strategy;

    if (overridden) {
      const newToolSequence = STRATEGY_TOOL_MAP[chosenStrategy] || plan.tool_sequence;
      const newSteps = newToolSequence.map((tool, i) => ({
        order: i + 1,
        tool,
        purpose: `LLM-selected: ${tool}`,
        driven_by_constraint_indices: plan.steps[0]?.driven_by_constraint_indices ?? [],
      }));

      const updatedPlan: MissionPlan = {
        ...plan,
        strategy: chosenStrategy,
        tool_sequence: newToolSequence,
        steps: newSteps,
        selection_reason: `LLM override: ${reasoning} (was: ${plan.strategy})`,
      };

      console.log(`[STRATEGY_REVIEWER] OVERRIDE: ${plan.strategy} → ${chosenStrategy} — ${reasoning}`);

      createArtefact({
        runId,
        type: 'diagnostic',
        title: `Strategy review: ${plan.strategy} → ${chosenStrategy}`,
        summary: reasoning,
        payload: {
          original_strategy: plan.strategy,
          original_tool_sequence: plan.tool_sequence,
          new_strategy: chosenStrategy,
          new_tool_sequence: newToolSequence,
          reasoning,
          raw_response: rawContent,
          latency_ms: latencyMs,
          tokens_in: usage?.prompt_tokens ?? null,
          tokens_out: usage?.completion_tokens ?? null,
        },
        userId,
        conversationId,
      }).catch(e => console.warn(`[STRATEGY_REVIEWER] Artefact failed: ${e.message}`));

      return { reviewed: true, overridden: true, plan: updatedPlan, reasoning, latencyMs };
    }

    console.log(`[STRATEGY_REVIEWER] CONFIRMED: ${plan.strategy} — ${reasoning}`);

    createArtefact({
      runId,
      type: 'diagnostic',
      title: `Strategy review: confirmed ${plan.strategy}`,
      summary: reasoning,
      payload: {
        original_strategy: plan.strategy,
        confirmed: true,
        reasoning,
        raw_response: rawContent,
        latency_ms: latencyMs,
        tokens_in: usage?.prompt_tokens ?? null,
        tokens_out: usage?.completion_tokens ?? null,
      },
      userId,
      conversationId,
    }).catch(e => console.warn(`[STRATEGY_REVIEWER] Artefact failed: ${e.message}`));

    return { reviewed: true, overridden: false, plan, reasoning, latencyMs };
  } catch (err: any) {
    const latencyMs = Date.now() - startMs;
    console.warn(`[STRATEGY_REVIEWER] LLM call failed: ${err.message} — keeping original plan`);
    return { reviewed: false, overridden: false, plan, reasoning: `Error: ${err.message}`, latencyMs };
  }
}

function buildPrompt(
  plan: MissionPlan,
  rawUserInput: string,
  intentNarrative: IntentNarrative | null,
): string {
  const constraintSummary = plan.constraint_mappings.map(m =>
    `- ${m.constraint_type}: field="${m.constraint_field}" value="${m.constraint_value}" hardness=${m.constraint_hardness}`
  ).join('\n');

  const narrativeSection = intentNarrative ? `
## Intent Narrative
Entity: ${intentNarrative.entity_description}
Key discriminator: ${intentNarrative.key_discriminator}
Findability: ${intentNarrative.findability}
Scarcity: ${intentNarrative.scarcity_expectation}
Exclusions: ${intentNarrative.entity_exclusions.join(', ')}
Suggested approaches: ${intentNarrative.suggested_approaches.join('; ')}` : '';

  return `You are the Wyshbone strategy reviewer. A rules-based planner has chosen a tool strategy for a user query. Your job is to review that choice and either CONFIRM it or OVERRIDE it with a better strategy.

## User Query
"${rawUserInput}"

## Constraints Identified
${constraintSummary || 'None beyond basic discovery'}

## Rules-Based Decision
Strategy: ${plan.strategy}
Tool sequence: ${plan.tool_sequence.join(' → ')}
Reason: ${plan.selection_reason}
${narrativeSection}

## Available Strategies (cheapest to most expensive)

1. **discovery_only** — Just search Google Places, return results. No verification.
   Use when: the query is simple discovery ("find pubs in Brighton"), no constraints need checking.

2. **discovery_then_direct_filter** — Search Google Places, then filter by name/field matching.
   Use when: constraints can be checked from the discovery data itself (e.g. "name contains X").

3. **discovery_then_rank** — Search Google Places, then rank/score results.
   Use when: the user wants "best" or "top" results, no hard verification needed.

4. **discovery_then_website_evidence** — Search Google Places, visit websites, extract evidence, verify with Tower. Most expensive.
   Use when: constraints genuinely require checking website content (e.g. "companies that offer X service" where X isn't in the Google Places listing).

## Decision Rules

- Prefer the CHEAPEST strategy that can satisfy the constraints
- If findability is "easy" or "medium" and constraints can be inferred from business type/category, discovery_only is often enough
- discovery_then_website_evidence should only be used when you genuinely cannot verify the constraint without visiting websites
- A status_check constraint does NOT automatically require website evidence — if the status is implied by the business category (e.g. "restaurants that serve food" — obviously they do), discovery_only is fine
- If scarcity_expectation is "scarce", consider whether expensive verification is worth it when there may be very few results to verify

Respond with JSON only:
{
  "strategy": "one of: discovery_only, discovery_then_direct_filter, discovery_then_rank, discovery_then_website_evidence",
  "reasoning": "2-3 sentences explaining your decision"
}`;
}
