Goal
Add a lightweight constraint checklist to Pass 1 of the mission extractor so the LLM explicitly classifies the types of constraints present in the user's request before producing the semantic interpretation.

This must NOT change the external architecture or the StructuredMission schema.

The checklist is purely an internal improvement to the Pass 1 reasoning step.

Current pipeline (do not change)

User input
→ Pass 1 semantic interpretation
→ Pass 2 structured mission
→ schema validation
→ mission planner
→ mission executor

We are modifying ONLY Pass 1.

PROMPT RULES FOR PASS 1

Update the LLM prompt so it explicitly instructs:

1. First classify which constraint categories appear in the user request.
2. Then write a clean semantic interpretation of the user's intent.

Important instructions:

• Interpret meaning, not phrasing
• Do not copy user phrasing
• Do not drop constraints
• Detect relationship language like:
  "works with", "partnered with", "supplies", "serves", "affiliated with"
• Detect ranking language like:
  "best", "top", "highest rated"
• Detect monitoring intent like:
  "keep checking", "alert me when"
