== WHAT TO CHANGE ==

Only ONE file changes: server/supervisor.ts

== THE PROBLEM ==

Right now, when the router returns SEARCH or ITERATE, it rewrites rawMsg and effectiveMsg to a clean query like "find pubs in Arundel", but then the OLD decision chain still runs:
- Message classifier (regex, redundant — router already classified)  
- Conversation state + phase detection (regex, redundant — router already decided)
- State-aware routing / result discussion / iteration (redundant — router already handled)
- Intent extractor shadow (LLM call, redundant — wastes tokens and 3-5 seconds)
- Cross-session vague reference resolver (regex, redundant — router already resolved)
- Constraint gate outer check (regex, redundant — router already confirmed searchable)
- Smart clarify quality check (LLM call, redundant — router already confirmed no clarification needed)

These add 10-30 seconds of unnecessary processing after the router has already made the decision.

== THE FIX ==

Add a boolean flag `_routerHandledSearch` that is set to true when the router successfully routes to SEARCH or ITERATE. Then wrap each redundant block in an `if (!_routerHandledSearch)` check so it's skipped.

== WHAT THIS ACHIEVES ==

When CONVERSATION_ROUTER_ENABLED=true AND the router returns SEARCH or ITERATE:
- Skips: message classifier, conversation state, state-aware routing, intent extractor shadow (saves 1 LLM call), cross-session vague resolver, INTENT_FIX, constraint gate, smart clarify quality check (saves 1 LLM call)

When CONVERSATION_ROUTER_ENABLED=false OR the router fails:
- Everything runs exactly as before. Zero changes to the fallback path.
