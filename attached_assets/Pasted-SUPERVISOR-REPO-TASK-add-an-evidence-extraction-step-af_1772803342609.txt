SUPERVISOR REPO TASK: add an evidence extraction step after WEB_VISIT so attribute queries produce strong evidence snippets, not just page titles.

Goal
For attribute queries like:
"Find cafes in Manchester that serve vegan food"

WEB_VISIT already fetches page text successfully.
But the current evidence artefacts are too weak because they mainly store shallow evidence like page titles, while the useful proof is buried inside text_clean.

We need a small evidence extraction step between:
WEB_VISIT
and
Tower judgement

Target architecture
1. Intent extractor understands the request
2. SEARCH_PLACES finds candidates
3. WEB_VISIT fetches website pages
4. NEW: evidence extraction step pulls out the best relevant snippets from page text
5. Tower judges those snippets against the original goal/constraint
6. Supervisor continues / stops / delivers

What to implement
1. After WEB_VISIT for attribute-style queries, inspect outputs.pages[*].text_clean
2. Extract 1 to 3 relevant snippets per lead for the requested attribute
3. Store them in a stronger attribute evidence artefact

The extracted evidence artefact should include at least:
- run_id
- lead_name
- lead_place_id
- attribute_raw
- attribute_key
- source_url
- page_url
- extracted_quotes: string[]
- page_title
- extraction_method
- investigation_strategy
- original_goal
- constraint_raw

Important
- Do not just store page titles as the main evidence
- Prefer sentence/snippet level evidence from text_clean
- Keep this deterministic and simple for now:
  - keyword-guided snippet extraction is fine
  - just pull the most relevant lines/paragraph fragments around likely matches
- If no relevant snippet is found, still store an evidence artefact saying no relevant snippet found

For this first version, use practical snippet extraction:
- split text_clean into sentences or short chunks
- rank chunks by relevance to attribute_raw / attribute_key
- store the top 1 to 3 chunks

Example
If WEB_VISIT text contains:
"Had a vegan brunch in Manchester at Pot Kettle Black"

the evidence artefact should store that as an extracted quote.

Success result
For:
"Find cafes in Manchester that serve vegan food"

the system should produce attribute evidence artefacts containing meaningful snippets like:
- "Had a vegan brunch in Manchester at Pot Kettle Black"
instead of weak evidence like only:
- "Page title: Our Menus"

Important constraints
- minimal production-safe change
- do not redesign Tower in this task
- do not reintroduce brittle semantic yes/no verdict logic in Supervisor
- Supervisor should extract evidence snippets, not make the final semantic judgement
