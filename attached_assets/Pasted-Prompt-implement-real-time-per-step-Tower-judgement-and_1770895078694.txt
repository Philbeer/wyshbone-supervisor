create a step_result artefact

react BEFORE continuing (RETRY / CHANGE_PLAN / STOP / CONTINUE)

emit AFR events for each stage so the UI shows Tower badges per step in order

This must be the primary execution path.

“Backfill” remains only as a safety net when a judgement is missing after completion.

Create step_result artefact

type: "step_result"

payload includes: goal, run_id, client_request_id, step_id, step_title, step_index, step_status, inputs_summary, outputs_summary, outputs_raw(optional), timestamps

Call Tower on step_result
Immediately after creating step_result:

call judgeArtefact() (tower-artefact-judge.ts)

send: { goal, success_criteria, artefact: step_result }

receive verdict with action: CONTINUE / RETRY / CHANGE_PLAN / STOP

Persist tower judgement for that step
Create a tower_judgement artefact with:

step_id, step_index

verdict + action + reasons + confidence + metrics

and a pointer to the judged artefact id

React inline

CONTINUE: proceed to next step

RETRY: retry the same step (increment attempt count, bounded)

CHANGE_PLAN: create a plan_update artefact (plan_v2), replace remaining steps and continue

step_completed

artefact_created(step_result)

tower_call_started

tower_verdict

artefact_created(tower_judgement)

supervisor_reaction_taken (continue/retry/change_plan/stop)

Remove reliance on backfill for normal runs

Ensure real-time step judgements happen for chat delegated runs and normal plan runs.

Backfill should only run if a run reaches terminal state and there is no tower_judgement at all.
