Fix 4.5: Pass 3 Example Contamination — False Clarification Triggers
Problem
The Pass 3 system prompt in server/supervisor/mission-extractor.ts has two examples. Example 2 is:
"find organisations that work with the local authority in Blackpool"
This is nearly identical to benchmark query B06 ("Find organisations that work with Blackpool council"). When B06 runs, the LLM pattern-matches it to this example and sets clarification_needed: true with a clarification question — instead of just running the search.
The clarification system was designed before the LLM intent extraction was good enough to handle ambiguous queries. Now, the 3-pass extraction pipeline (Pass 1 semantic → Pass 3 intent narrative → Pass 2 structured) is sophisticated enough that almost every query can be executed without clarification. The only queries that genuinely need clarification are ones like "find pubs in Narnia" (fictional location) or truly incoherent inputs.
