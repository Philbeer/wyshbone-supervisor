Fix delivery_summary truth when CVL verification exists

Context
We now emit verification_summary artefacts with fields like:

verified_exact (or verified_exact_count)

requested_count_user

hard_unverifiable constraints (e.g. beer garden)

hard_unknown_count

We also correctly end the run with verdict/action like:

verdict=stop OR verdict=change_plan (truthful)

Bug
Even when:

verification_summary says verified_exact=0

and the final run outcome is stop/change_plan
…the delivery_summary artefact still emits:

title/status = PASS

exact = raw leads count (e.g. 20)

shortfall = 0
This is lying.

Requirements

If a verification_summary exists (CVL present):

delivery_summary.exact MUST equal the verified exact count:
exact = verification_summary.verified_exact ?? verification_summary.verified_exact_count ?? 0

requested MUST be requested_count_user (from constraints_extracted / verification_summary), default 0 if missing

shortfall = max(0, requested - exact)

cvl_verified must be set to exact

delivery_summary status/title MUST align with final outcome:

If final verdict/action is STOP -> delivery_summary MUST be STOP (never PASS)

If final verdict/action is CHANGE_PLAN -> delivery_summary MUST be CHANGE_PLAN / NEEDS_VERIFICATION (never PASS)

PASS is only allowed when:

CVL exists AND exact >= requested_count_user AND final verdict is not stop/change_plan

(or legacy path: no CVL -> keep existing behavior)

Backwards compatibility:

If no verification_summary exists, keep current legacy behaviour unchanged.

Implementation notes

Do NOT touch Tower.

Make minimal changes in delivery-summary.ts and/or the Supervisor call site that constructs/emits the delivery_summary artefact.

Ensure delivery_summary includes a clear stop_reason when STOP/CHANGE_PLAN, ideally referencing “unverifiable hard constraint: <attr>” when present.

Acceptance test (run exactly one)
Prompt: “Find 7 pubs in Chichester that have a beer garden”
Expected:

verification_summary: verified_exact=0, requested=7, hard_unverifiable includes beer garden

final verdict/action: stop OR change_plan

delivery_summary MUST NOT say PASS

delivery_summary.exact=0, shortfall=7, cvl_verified=0

delivery_summary.stop_reason mentions unverifiable hard constraint (beer garden)
