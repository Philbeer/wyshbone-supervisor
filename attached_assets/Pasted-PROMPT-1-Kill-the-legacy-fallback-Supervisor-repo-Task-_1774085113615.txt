## Task: Remove legacy execution fallback — replace with honest error

The Supervisor currently has two execution paths in `processChatTask` inside `server/supervisor.ts`:

1. **Mission path** (`executionSource === 'mission'`) — calls `executeMissionWithReloop()` — this is the active, correct path
2. **Legacy fallback** (`executionSource === 'legacy'`) — calls `this.executeTowerLoopChat()` — this is old code that delivers unverified junk results

We are removing the legacy fallback entirely. When mission extraction fails, we return an honest error instead of degraded results.

**Replace the else branch** so that instead of calling `executeTowerLoopChat`, it throws an error that will be caught by the existing `catch (execErr)` block immediately below:
