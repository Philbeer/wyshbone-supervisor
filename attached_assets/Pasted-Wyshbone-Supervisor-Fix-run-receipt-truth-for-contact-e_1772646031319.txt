Wyshbone Supervisor — Fix run_receipt truth for contact extraction and artefact discovery

Problem

The run_receipt currently reports incorrect contact extraction statistics.

Example:
run_receipt shows

contact_extraction_attempted_count: 0
contact_sources_used: ["lead_pack"]

However AFR shows multiple CONTACT_EXTRACT tool calls during the run.

This means the receipt builder is not correctly discovering or counting CONTACT_EXTRACT artefacts.

The receipt must reflect what actually happened in the run.

Goal

Make run_receipt a truthful summary of contact extraction activity.

Required behaviour

1) CONTACT_EXTRACT artefact detection

Find where run_receipt is built.

Update artefact discovery so that CONTACT_EXTRACT artefacts are detected correctly.

Possible causes to investigate:
- case mismatch in artefact type
- wrong artefact filter
- incorrect artefact payload path
- filtering only certain artefact categories

Ensure CONTACT_EXTRACT artefacts are included when scanning artefacts for receipt statistics.


2) Correct contact_extraction_attempted_count

Set contact_extraction_attempted_count to the number of leads where a CONTACT_EXTRACT attempt occurred.

Rules:
- If a CONTACT_EXTRACT artefact exists for a lead, count it as attempted.
- Count attempts even if no email or phone was found.


3) Correct contact_sources_used

Populate contact_sources_used dynamically.

Rules:

Include "contact_extract" if any CONTACT_EXTRACT artefacts exist.

Include "lead_pack" if contact information was extracted directly during lead_pack.

Examples:

Case A:
lead_pack extracted contacts only

contact_sources_used: ["lead_pack"]

Case B:
contact_extract tool used

contact_sources_used: ["lead_pack","contact_extract"]

Case C:
contact_extract only

contact_sources_used: ["contact_extract"]


4) Do not change the structure of run_receipt

Keep the same schema and fields.

Only correct the counting and source detection.


5) Acceptance criteria

Run the query:

"Find pubs in Arundel that mention live music on the website"

Expected result:

contact_extraction_attempted_count > 0

contact_sources_used includes "contact_extract"

run_receipt must match AFR activity.


Deliverables

Return:
- files changed
- explanation of the bug that caused CONTACT_EXTRACT artefacts to be missed
- before/after example run_receipt
