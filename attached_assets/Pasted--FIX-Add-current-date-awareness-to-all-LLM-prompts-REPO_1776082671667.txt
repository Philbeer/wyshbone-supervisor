# FIX: Add current date awareness to all LLM prompts

## REPO: wyshbone-supervisor-Post-CC

## THE PROBLEM

The LLM models used (claude-3-5-haiku, gpt-4o-mini) have training cutoffs around early-mid 2024. They don't know today's date, who the current PM is, or any recent events. This causes temporal verification to fail ("opened in 2023" accepted as "recent") and chat responses to give outdated information.

## THE FIX

Add a dynamic date/context preamble to the key system prompts. This is a simple string prepended to existing prompts — no logic changes.

### Step 1: Create a shared context utility

Create a new file `server/supervisor/current-context.ts`:

```typescript
/**
 * Dynamic context preamble for LLM prompts.
 * Provides current date and basic facts so models with older training cutoffs
 * can reason about time and avoid confidently stating outdated information.
 */

export function getCurrentDatePreamble(): string {
  const now = new Date();
  const readable = now.toLocaleDateString('en-GB', { 
    weekday: 'long', day: 'numeric', month: 'long', year: 'numeric' 
  });
  const iso = now.toISOString().split('T')[0];
  const oneYearAgo = new Date(now.getTime() - 365 * 24 * 60 * 60 * 1000)
    .toISOString().split('T')[0];
  const currentYear = now.getFullYear();

  return `TODAY'S DATE: ${readable} (${iso}). Current year: ${currentYear}. "Last 12 months" = after ${oneYearAgo}.`;
}

export function getCurrentContextPreamble(): string {
  const datePreamble = getCurrentDatePreamble();
  const currentYear = new Date().getFullYear();

  return `${datePreamble}

IMPORTANT: Your training data may be outdated. If asked about current events, leaders, or recent developments, acknowledge that your information may not be current and suggest the user verify. Do not confidently state facts about events after early 2024.

Basic current context (${currentYear}):
- UK Prime Minister: Keir Starmer (Labour, since July 2024)
- US President: Donald Trump (since January 2025)
- UK currency: GBP. EU currency: EUR.
- This app (Wyshbone) is a B2B lead generation tool based in the UK.`;
}
```

## WHAT THIS ACHIEVES

- **Temporal verification**: "Opened in May 2023" is correctly rejected for "last 12 months" queries because the LLM knows today is 2026
- **Chat responses**: "Who's the PM?" gets "Keir Starmer" instead of an outdated answer or "I don't know"
- **Synonym expansion**: Generates "est. 2026" and "est. 2025" instead of "est. 2024"
- **General awareness**: The LLM knows the current year for any date-related reasoning

## UPDATING THE CONTEXT

The `getCurrentContextPreamble()` function contains a small block of basic facts (PM, President). This should be updated when major changes happen (elections, etc.). It's one file, one function — easy to maintain.

## TESTING

2. Chat: "who is the UK prime minister?" — should say Keir Starmer
3. Search: "micropubs that opened in the last 12 months in Sussex" — should reject 2023 openings
4. Search: "pubs in Arundel" — should still work (date doesn't affect non-temporal queries)

## CRITICAL RULES

1. Use `new Date()` for all date calculations — never hardcode dates
2. The basic facts block (PM, President) IS hardcoded — that's intentional, update it when needed
3. Keep the preamble SHORT — it's prepended to every prompt, don't add essays
4. The date preamble goes on ALL prompts; the full context (with PM etc.) only goes on the chat handler
5. Do NOT change any prompt logic — just prepend the context string
