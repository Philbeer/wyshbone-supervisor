GOAL: Fix the constraint-to-claim helper so negation operators render as POSITIVE claims for Tower verification. Currently they render as negative ("does NOT mention X") which Tower can't verify (you can't prove an absence). Result: Tower returns no_evidence for every lead, gateway's STRICT flip lets all leads through regardless of whether they actually violate the negation.

Architectural intent: Tower verifies POSITIVE claims. Gateway flips the verdict for negated constraints (existing STRICT flip in finalize-delivery.ts). The claim helper needs to render positive, not negative.

In renderConstraintAsClaim, change the negation-operator branches to render the SAME claim as the positive form of that operator. Specifically:

text_compare:
- not_contains → "the {field} contains '{value}'" (same as contains)

attribute_check:
- not_has → "they have {value}" (same as has)

- not_equals → "they are currently {value}" (same as equals/has)

Simplest implementation: at the top of the function, detect negation by checking if operator starts with "not_". If yes, strip the "not_" prefix and dispatch to the positive-operator rendering branch (so you don't duplicate logic). The string "not_" itself never reaches the rendered output.
