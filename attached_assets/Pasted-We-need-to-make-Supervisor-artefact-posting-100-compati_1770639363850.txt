We need to make Supervisor artefact posting 100% compatible with the UI’s /api/afr/artefacts endpoint and ensure identifiers match real UI runs.

Truths:

UI expects POST /api/afr/artefacts with runId + type + payload (and optional createdAt).

UI also now supports payloadJson, but we should still send the canonical format to reduce risk.

“View results” should work by runId, so the runId Supervisor uses must match the run_id provided by the UI in supervisor_tasks.request_data.

Implement these production-grade changes:

Ensure Supervisor always uses UI-provided IDs:

In processChatTask, set:

chatRunId = requestData.run_id (required for chat tasks) else fallback chat_${task.id}

clientRequestId = requestData.client_request_id (optional)

If requestData.run_id is missing, emit a clear AFR event (or log) so we can see that the UI didn’t include it.

Post artefacts using the UI canonical body shape:
POST to ${UI_URL}/api/afr/artefacts with JSON body:
{
runId: chatRunId,
clientRequestId,
type: "leads",
payload: {
title: "...",
summary: "...",
leads: [...normalized leads...],
query: {...},
tool: "SEARCH_PLACES"
},
createdAt: new Date().toISOString()
}
Do NOT send payloadJson unless necessary. Keep it simple.

Keep the gating rule:

Only emit artefact_created and run_completed after the POST returns 200 and we parse artefactId from the response.

If POST fails, emit a single AFR event that says artefact_post_failed with the HTTP status code (no secret URL, no stack trace spam).

Normalize leads consistently:
Each lead object should include at minimum:

name

address (string)

phone (string or null)

website (string or null)

placeId

source: "google_places"

score (if available) else null

Add minimal observability:

Log one line: “Posting artefact to UI: runId=<...> status=<...>”

Do not log PII-heavy payloads.

Deliverable:

Update supervisor.ts and any helper used for postArtefactToUI.

Restart and run one real chat task to verify we get 6 events: mission_received, router_decision, tool_call_started, tool_call_completed, artefact_created, run_completed.
