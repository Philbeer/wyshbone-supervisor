Logic:
1. Query `loop_state` for the given `run_id`, ordered by `loop_number ASC`. If no rows or Supabase error, return null.
2. Find all "fully completed" rows ÔÇö those where `completed_at IS NOT NULL` and `status` is not 'active' (or where judge_verdict has a verdict field).
3. Look at the LAST row in the result set to determine crash point:

   **Case A ÔÇö Last row is fully completed (has completed_at) and gate_decision.decision === 're_loop':**
   The system completed a loop but crashed before starting the next one.
   Ôćĺ `resumeFrom: 'planner'`, `lastCompletedLoop: lastRow.loop_number`

   **Case B ÔÇö Last row is fully completed and gate_decision.decision === 'stop_deliver':**
   The run actually finished its loop phase but crashed during combined delivery.
   Ôćĺ `resumeFrom: 'planner'` is wrong here ÔÇö but for MVP we'll just return null (don't resume, let it re-run). Add a comment: `// TODO: resume from combined delivery phase`

   **Case C ÔÇö Last row has `executor_completed === true` but `completed_at IS NULL` (judge never ran):**
   The executor finished but the judge/gate didn't run.
   Ôćĺ `resumeFrom: 'judge'`, `lastCompletedLoop: lastRow.loop_number - 1`
   Ôćĺ Set `lastExecutorOutput` by parsing `lastRow.executor_output_full`
   Ôćĺ Set `lastPlannerDecision` by parsing `lastRow.planner_decision`

   **Case D ÔÇö Last row has `executor_completed === false` or missing (executor didn't finish):**
   Ôćĺ `resumeFrom: 'planner'`, `lastCompletedLoop: lastRow.loop_number - 1`
   (This means: redo the current loop from scratch, but keep all previously completed loops)

4. Rebuild state from completed rows:
   - `accumulatedEntities`: Parse the `accumulated_entities` JSON from the last fully-completed row (or from the phase-1 write of the current incomplete row if Case C). Parse as `ExecutorEntity[]`.
   - `loopHistory`: Build `LoopRecord[]` from the fully completed rows. Each row has `planner_decision`, `executor_output_full` (parse to get the executorOutput), `judge_verdict`, `gate_decision`, `created_at`, `completed_at`, duration = Date.parse(completed_at) - Date.parse(created_at).
     For the gate_decision, reconstruct the full GateDecision shape ÔÇö the persisted version is compact (decision, loopNumber, circuitBreaker, failureContext, suggestedNextExecutor). Set `contextForward.accumulatedEntities` to an empty array (it's only used during live runs, not for history), `contextForward.loopHistory` to empty array, and `contextForward.variableState` from the judge_verdict's variableState.
   - `executorsTriedSoFar`: Map completed rows to `row.executor_type`. If Case C, also include the incomplete row's executor_type.
   - `chainId`: From the first row's `chain_id`.
   - `finalRawResult`: From the last completed row's executor_output_full.rawResult (if available), or empty object.

5. Log: `[RECOVERY] checkForResumableState runId=${runId}: found ${rows.length} loop_state rows, resumeFrom=${checkpoint.resumeFrom}, lastCompletedLoop=${checkpoint.lastCompletedLoop}`

6. Return the ResumeCheckpoint.
