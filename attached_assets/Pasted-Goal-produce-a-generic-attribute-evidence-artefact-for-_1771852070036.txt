Goal: produce a generic attribute_evidence artefact for any HAS_ATTRIBUTE constraint using existing WEB_SEARCH / WEB_VISIT outputs.

Paste to CC in the Supervisor repo:

You are working in Wyshbone Supervisor.

Problem

CVL currently marks many attributes as unknown because Places API cannot verify them, even when we already have strong web evidence from WEB_SEARCH / WEB_VISIT. Example: “live music” for The Red Lion Arundel.

Required change (general, not curve-fit)

For any constraint with constraint_type = HAS_ATTRIBUTE (eg c_attr_live_music, beer_garden, dog_friendly, etc.) implement an evidence collection step that produces a new artefact per lead+attribute:

Artefact type: attribute_evidence
Key fields:

run_id, lead_place_id, lead_name

attribute_key (eg "live_music")

verdict: yes | no | unknown

confidence: high | medium | low

evidence: {source_url, quote, source_type} (and link to existing evidence ids if you have them)

rationale (1 sentence)

Evidence sources (use what we already have)

Prefer WEB_VISIT pages for the official site if available (look for keyword matches on-page).

Fallback to WEB_SEARCH snippets if WEB_VISIT didn’t crawl or no pages match.

If nothing found, verdict stays unknown.

Trust ladder (generic)

Official site page with clear mention -> verdict yes, confidence high

Reputable directory / local guide with clear mention -> verdict yes, confidence medium

Only weak hints -> unknown, confidence low

Contradiction from official site (eg “no live music”) -> verdict no, confidence high

Keyword matching (generic)

Implement a simple mapping from attribute_key -> keyword set.
Start with a minimal safe set:

live_music: ["live music", "live band", "bands", "open mic", "gigs", "gig", "music listings"]
(Do not hard-code The Red Lion or any domain.)

Output requirement

Write attribute_evidence artefacts for each lead that is being verified for HAS_ATTRIBUTE constraints, so Tower can consume them.

Acceptance test

Run: “Find 5 pubs in Arundel with live music”
Expect: For The Red Lion, an attribute_evidence artefact exists with verdict yes and a supporting quote + source_url (from redlionarundel.com or VisitArundel etc).

Keep this change minimal and production-grade. Do not change unrelated plan execution.
