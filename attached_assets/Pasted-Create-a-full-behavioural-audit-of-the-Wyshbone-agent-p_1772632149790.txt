Create a full behavioural audit of the Wyshbone agent pipeline.

Do NOT modify code. Report only.

Produce a document:

REPORT_SYSTEM_BEHAVIOUR_AUDIT.md

Goal:
Understand how the system currently handles real user inputs across the entire pipeline.

This report must analyse the following architecture layers:

Input parsing

Constraint extraction

Hard vs soft constraint classification

Verifiability classification

Constraint Verification Layer (CVL)

Capability checks

Planner behaviour

Tool selection

Replan policy

Tower judgement behaviour

STOP behaviour

Clarify gate behaviour

Artefact generation

Run receipt generation

UI truth alignment

The report should explain what the system currently does, not what it should do.

Explain the full execution pipeline step by step:

User input

→ Capability check
→ Plan generation
→ Tool execution
→ Artefacts created

→ Replan or stop
→ Run receipt

• file paths
