## Task: Replace GPT-4o fallback VERIFIED/UNVERIFIED/CONTRADICTED prefix parsing with structured JSON output

The current GPT-4o fallback asks the LLM to start with VERIFIED/UNVERIFIED/CONTRADICTED, but GPT-4o interprets "VERIFIED" as "I verified the facts" not "the constraint is true." This causes false positives no matter how many contradiction detectors we add. The fix: ask for structured JSON with separate fields for "did you find the business" and "does it match the constraint."

In server/supervisor/mission-executor.ts, find the GPT-4o fallback section (search for GPT4O_FALLBACK).

### What this achieves

Before: "Start with VERIFIED/UNVERIFIED/CONTRADICTED" → GPT-4o confuses "I verified the info" with "the constraint is true" → regex contradiction detectors as safety nets → still fails on markdown formatting

After: "Return JSON with business_found and constraint_met as separate fields" → GPT-4o correctly sets business_found=true, constraint_met=false for Two Fifteen → no ambiguity, no regex needed

The legacy prefix parsing is kept as a fallback only if JSON parsing fails, which handles any edge case where GPT-4o ignores the JSON instruction.

### What gets removed

The contradictionSignals array and hasNegatedConstraint check are no longer needed for the main path — they only exist in the legacy fallback now. They can be cleaned up in a future prompt but are harmless to leave.
